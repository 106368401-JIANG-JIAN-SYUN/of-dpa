Introduction

This document intends to describe some of the key advantages of an OF-DPA based OpenFlow 1.3 agent over the earlier OpenFlow 1.0 protocol based agents. A comparative analysis of both these agents is also done to highlight these benefits. The document then describes a use case to demonstrate a Traffic Engineering solution that illustrates these benefits.

This document describes a configuration that is suitable for testing the controller/agent interaction during engineering development, and a test case that can be used by QA for final testing. This is an engineering document that includes a lot of detail. This same TE framework has also been leveraged to produce a Traffic Engineering Demonstration that can be used with customers and at trade shows. For details on this please see the "OF-DPA 1.0 Traffic Engineering Demo" document.
