Porting a New Platform

This section provides the steps required to add support for a new platform in OF-DPA.

New Platform Build Directory

Do the following changes in the output build folder:

  • Add a new directory in the output folder with the name <platform>
  • Copy the Makefile from the existing output/gto-trident folder to this new <platform> folder
  • Modify the variables in the new Makefile as appropriate for the new platform and for the build environment. The variables most likely to need modifying are:
    • CROSS_COMPILE
    • LIBPTHREAD
    • KERNDIR
    • platform: This is the name of the platform which SDK uses, not the name of the OF-DPA platform. There should be a directory by this name in ofdpa/sdk/systems/linux/user

The new platform is ready to be built.

Note: The two <platform> directories created above (in the systems/platform/ and output/ directories) must have the same name. However, they do not have to match the platform name used by the SDK.
